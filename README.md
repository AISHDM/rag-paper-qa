# Research Paper Q&A — RAG System

Ask natural language questions over research papers and get grounded answers with source citations. Built with LangChain, ChromaDB, Groq (Llama 3), and FastAPI.

## Overview

This project implements a **Retrieval-Augmented Generation (RAG)** pipeline:

```
PDF Papers → Chunking → Embedding → ChromaDB
                                        ↓
User Question → Similarity Search → Relevant Chunks → Llama 3 → Answer
```

Instead of relying on an LLM's training data, the system retrieves relevant text directly from your documents before generating an answer. If the answer isn't in the papers, it says so — no hallucination.

## Architecture

| Component | Tool | Purpose |
|---|---|---|
| PDF Loading | LangChain PyPDFLoader | Extract text from papers |
| Chunking | RecursiveCharacterTextSplitter | Split into 500-char chunks |
| Embeddings | all-MiniLM-L6-v2 (HuggingFace) | Encode chunks as vectors |
| Vector Store | ChromaDB | Store and search embeddings |
| LLM | Llama 3.1 8B via Groq | Generate grounded answers |
| API | FastAPI | Serve the pipeline over HTTP |

## Setup

```bash
git clone https://github.com/AISHDM/rag-paper-qa.git
cd rag-paper-qa

conda activate ml-chem
pip install -r requirements.txt
```

Create a `.env` file with your Groq API key (free at console.groq.com):

```
GROQ_API_KEY=gsk_your_key_here
```

## Usage

**Step 1: Ingest papers into the vector database**

```bash
python src/ingest.py
```

**Step 2: Start the API**

```bash
uvicorn src.api:app --reload --port 8000
```

**Step 3: Ask questions**

```bash
# Ask a question
curl -X POST "http://localhost:8000/ask" \
  -H "Content-Type: application/json" \
  -d '{"question": "How does the attention mechanism work?"}'

# List indexed papers
curl http://localhost:8000/papers
```

Or open `http://localhost:8000/docs` in your browser for the interactive API explorer (auto-generated by FastAPI).

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | Health check |
| POST | `/ask` | Ask a question |
| GET | `/papers` | List indexed papers |

**POST /ask request body:**
```json
{
  "question": "What is self-attention?",
  "k": 4
}
```
`k` controls how many chunks are retrieved (default: 4).

## How RAG Works

**Without RAG:** LLM answers from training data → may hallucinate or give outdated answers.

**With RAG:**
1. Your question is converted to a vector (embedding)
2. ChromaDB finds the most similar chunks from your papers
3. Those chunks are passed to the LLM as context
4. The LLM answers using only that context

The result: answers grounded in your actual documents, with citations.
